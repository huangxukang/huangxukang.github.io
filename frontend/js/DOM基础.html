# DOM基础

概念：document object model文档对象模型，主要的目的就是把网页里面的元素当成对象一样操作

网页当中的元素与我们之前学习的对象是非常像的，它有标签名（对象名）、属性、也有属性值

把网页当中的元素当成对象一样去操作的时候，这样就方便了我们对网页的操控

DOM的本质是网页与JS的结合 ，它将JS应用在了网页上，用JS来操作网页

DOM里面又分为两个部分，第一个部分是JS与HTML的结合，第二部分是JS与CSS的结合

  1. 整个页面最终会形成一个document对象
  2. 页面上面所有的元素（标签）最终都会转换成JS里的对象
  3. 如果是非IE的浏览器，它会根据每ID生成一个对象方便直接取用(window.id)

# 通过JS获取页面元素

当前页面指是document

<div id="div1">第一个盒子</div>
<div class="div2">第二个盒子</div>
<div class="div2">我是第二个盒子的副本</div>
<p>我是第三行</p>
<input type="radio" name="sex">男
<input type="radio" name="sex">女

  1. 通过ID来进行获取页面元素

    document.getElementById("div1");

    通过这各方式去获取，永远只能够获取到0个(null)或1个，如果找到了就是1个，如果找不到就是0个

    注意ID名重复的情况，找到的永远是第一个

  2. 通过class名称来获取页面元素

    document.getElementsByClassName("div2")';'

    通过这种方式，获取的是一个HTMLCollection的集合（类数组），具备数组的特性，不具备数组的方式，这个集合里面存放的就是DOM对象，如果一个元素都获取不到，它就是一个空的类数组[]

  3. 通过标签名去获取页面元素

    document.getElementsByTagName("p");
document.getElementsByTagName("div")

    通过这种方式，获取到的也是一个HTMLCollection，通过页面上面的某一些标签来获取某一个元素

  4. 通过name属性去获取页面元素

    document.getElementsByName("sex");

    它返回的是一个NodeList的集合（类数组）,具备数组特性，不具备数组方法，这个集合里面存放的就是DOM对象，如果一个元素都获取不到，它就是一个空的类数组[]

上面的4种方式是最基本的4种方式，这种方式没有任何兼容性，可以解决日常工作中的很多问题

<div id="div1">
	<div class="div2">
        我爱北京天安门
    </div>
    <div class="div2">
        天安门上太阳升
    </div>
    <div class="div3">
        大家好才是真的好
    </div>
</div>
<div class="div2">第二个盒子</div>
<div class="div2">我是第二个盒子的副本</div>
<p>我是第三行</p>
<input type="radio" name="sex">男
<input type="radio" name="sex">女

在上面的网页结构里面，如果想获取div1里面的div2又怎么办

var div1=document.getElementById("div1");  
//第一步：先获取div1的元素
div1.getElementsByClassName("div2");  
//第二步：再到div1里面去找class="div2"的元素

这是我们很早以前的一种解决方式 ，现在，不再这样使用，使用新的方案（强列推荐）

新的方案是结合的后期出现的HTML5与CSS3所出来的东西，CSS已经出了9大选择器了，那么我们能不能把这些应用到JS上面的呢

  1. 通过querySelector()来选择元素

    document.querySelector("#div1");  //id="div1"
document.querySelector(".div2");   //class="div2"的第一个元素

    这种方式是通过选择器来选择元素，如果找到了元素就返回1个元素，如果找不到就返回null

  2. 通过querySelectorAll()来选择元素

    document.querySelectorAll("#div1");
document.querySelectorAll(".div2");
document.querySelectorAll("#div1>.div2")

    它返回是一个NodeList集合，里面包含了要查找的元素，找到了就返回，如果找不到就返回一个空的NodeList[]

总结： getElementById和querySelector都是返回一个或0个，如果找到就返回一个，如果找不到就返回空值

querySelectorAll和后面的getElementsBy***返回的是一个集合

小技巧：后期会经常获取页面上面的元素，把上面的querySelectorAll的方法封装起来

// 给一个选择器，返回一个选中的元素的集合
function $(selector){
    return document.querySelectorAll(selector);
}

# DOM结构

DOM本身是一个对象，现在来分析一个对象

通过对三种标签的对象，最终发现，所以的元素最终都会继承自Node(EventTarget暂时先不做考虑)，可以把Node对象做为所有文档对象（页在元素）最终的父级，这个时候，只需要把Node对象弄清楚了，那么，所有的元素也都弄清楚了

# Element与Node的区别

  1. 在网页当中，所有的标签它都是一个元素(Element)，同时也是一个节点(Node)
  2. 但是并不是所有的节点Node都是元素Element（节点包括元素，文字，注释，回车）
<div id="div1">
    我在前面插入了文字
    <div class="div2">第一行</div>
    <div class="div2">第二行</div>
    <!-- 这是一个注释 -->
</div>

在上面的网页结构里面，div1当中的元素就仅仅只有class="div2"的两个标签，但是如果是节点Node，则包含了注释与元素，还有回车键

# Element常用属性

  1. children属性，获取当前元素下面的子级元素，它是一个HTMLCollection的集合

    这个时候，获取页面子级元素就已经有三各方式了

    document.getElementById("div1").getElementsByClassName("div2")[0]
document.querySelector("#div1>.div2:first-child")
div1.children[0]

  2. parentElement属性，用于获取当前元素的父级元素

  3. nextElementSibling属性，用于获取当前元素的下一个兄弟元素

  4. previousElementSibling属性，用于获取当前元素的上一个兄弟元素

  5. className属性，获取或设置当前元素的class名称

    <style>
	.p2{        color: red;    }
    .p3{        color: blue;    }
    .p4{        font-size: 56px;    }
</style>
<body>
     <p id="p1" class="p2">第二个元素，我是p</p>
</body>
<script>
	// 在p1标签上面追加p3
    p1.className=p1.className+" p3";
    // 希望再把p3去掉
    p1.className=p1.className.replace("p3","");
</script>

    通过这种去操作class非常不方便，它使用的是字符串的方式去操作，非常麻烦

  6. classList属性，返回当前元素的class名称集合，它返回的是一个DOMTokenList对象（类数组），这个集合里面包含了当前所有class样式名称

    • add()方法，在当前的classList里面添加一个class名称
    • remove()方法，在当前的classList里面移除一个class名称
    • toggle()方法，如果你有这个样式 ，就删除，如果没有，就添加
    • contains()方法，判断当前的class集合里面有没有这个class样式，true代表存在，false代表不存在

  7. firstElementChild属性，获取当前元素下面的第一个子元素，相当于children[0]

    获取第一个元素还可以通过document.querySelector("#div1>.div2:first-child");

  8. innerText属性，获取或设置当元素里面的文本内容，如果你赋值的时候是HTML标签，它不会进行转义

    <p id="p1">
    <b>我是p标签</b>
</p>
<script>
	var p1=document.querySelector("#p1");
    p1.innerText;  //取值
    p1.innerText="我在向你赋值";
    p1.innerText="<a href='#'>百度一下</a>";   //赋值进去的依然是一个文本
    p1.innerHTML;  //拿到html标签和文本
</script>

  9. innerHTML属性，获取或设置当前网页的标签内容 ，如果赋一个HTML的文字 ，它会进行网页的转义

  10. nodeType属性，返回当前元素的节点信息，如果是元素则返回1

  11. tagName属性，返回当前元素的标签名（全大写）

# Element常用方法

  1. createElement()，根据标签创建一个元素

    var div1 = document.createElement("div");
var p1=document.createElement("p");

  2. appendChild()，向当前元素里面的最后去追加一个新的元素

    div1.appendChild(p1); // 把p1追加到div1内部的最后面去

  3. remove()删除当前元素（里面的子元素也会没有）

  4. removeChild()删除指定的子元素

    // 删除子元素其实本质上面是有两个方法的
div1.children[1].remove(); // 直接找到子元素去删除
div1.removeChild(div1.children[1]); // 删除指定的子元素

  5. cloneNode()克隆一个相同的元素出来，它接收一个参数，这个参数是布尔类型，默认是false，如果给一个true，则代表要克隆子元素

    var dd=div1.cloneNode();  //只克隆当前元素

var dd1=div1.cloneNode(true);  //克隆当前的元素和所有的子元素

    注意：cloneNode()方法无论接不接true的参数，拷贝出来的都是深拷贝

  6. insertAdjacentElement()，向指定位置插入指定元素

    <ul id="ul1">
    <li>第一项</li>
    <li>第二项</li>
    <li>第三项</li>
</ul>
<script>
	var ul1=document.querySelector("#ul1");
    //创建一个新的li
    var newLi=document.createElement("li");
    newLi.innerText="插入项";
    newLi.classList.add("addli");
    ul1.insertAdjacentElement("位置",newLi);
    //它的第一个参数是一个特殊的参数，它代表要插入的位置，有以下四个值
    ul1.children[1].insertAdjacentElement("afterEnd",newLi);
</script>
    参数值 解释
    beforeBegin 开始之前
    afterBegin 开始之后
    beforeEnd 结束之前
    afterEnd 结束以后

    除了有insertAdjacentElement插入元素以外，还有insertAdjacentHTML和insertAdjacentText这两个方法，只是一个是插入网页文字，一个是插入文字

    var newLi="<li class='addli'>插入项</li>"; // 网页HTML的字符串，它不是一个元素
 ul1.insertAdjacentHTML("beforeBegin",newLi);
// ------
var ul1=document.querySelector("#ul1");
var str="hello world";
ul1.insertAdjacentText("beforeEnd",str);

    针对插入的方式不同，可以调用不同的方法来进行，接下来的方法是对DOM对象的属性进行操作

  7. 标签自己所具备的属性，我们应该怎么操作呢？

    可以利用对象基本知识对象.属性名或对象["属性名"]来进行操作

    userName.type;
userName.placeholder;
a1.href;
a1.target;

    上面是取值过程，如果要赋值则直接加一个等号赋值就可以

    userName.type="password";

  8. 标签自定义属性，又如何去操作呢?

    一般情况下，标签可以随便自定义属性（只要不与系统的属性重名，均可）,但是，工作当中，W3C推荐在自定义的属性前面添加一个data-

    // 现在，写一个自定义的属性
<input id="input2" type="text" data-biaogege="标哥哥" data-age="18" />

    网页在渲染的时候，发现了以data-开头的属性，那么它会把当前元素上面所有以data-开头的属性封装一个对象，这个对象的类型是DOMStringMap类型，然后把这个生成好的对象命名成dataset，放在当前的元素上面做为一个属性

    input2.dataset;  //获取所有以data-开头的属性
input2.dataset.biaogege;  //获取具备的属性值 
input2.dataset.age;

    注意事项

    一般情况下，不建议在自定义属性data-后面写成驼峰命名，如下

    data-userName;
data-girlAge;

    <div id="div1" data-userName="杨标" data-boyAge="18"></div>

    生成的标签如下

    它把大写变成了小写，所以再取值的时候

    div1.dataset.username;
div1.dataset.boyage;

    但是: 如果非要写成驼峰的，应该像下面这种方式来写

    <div id="div1" data-userName="杨标" data-boyAge="18"></div>

<div id="div2" data-user-name="杨标" data-boy-age="18"></div>

    这个时候，生成的dataset如下图所示

    这个时候生成的dataset就变成驼峰命名了

  9. getAttribute()可以获取任何方式定义的属性

    input2.getAttribute("type");
input2.getAttribute("data-biaogege"); // 这个地方不能省略掉data

  10. setAttribute()设置任何方式定义的属性值

    input2.setAttribute("type","text"); // 前面一个参数代表属性名，后面一个参数代表属性值
input2.setAttribute("data-biaogege","黄淑玲");

    赋值的时候，第一个参数代表属性名，第二个参数代表属性值

  11. removeAttribute()删除任何一个属性

以下几个属性比较特性，它特殊的地方在属性值上面，主要体现在单属性上面，目前所接触到的单属性，readonly，disabled，selected，checked，loop，autoplay

单属性返回的值要么是true，要么是false，如果是true则代单属性成立，如果是false则代表单属性不成立

<input type="checkbox" id="ck1">
<input type="text" readonly value="123123" id="userName">
<script>
	ck1.checkecd=true;   //设置为选中状态
    userName.readonly=false;  //将readonly只读设置为false
</script>
  1. hasAttribute()判断当前元素有没有这个属性，有就返回true,没有就返回false

# DOM基础补充

网页当中有一些特殊的标签会生成一些特殊的属性

  1. document.documentElement指的是html标签生成的对象
  2. document.title指的是网页的标题title标签生成的对象
  3. document.body指的是网页的body标签生成的对象
  4. document.links代表网页link标签所生成的对象的集合，它是一个HTMLCollection

# Node节点补充

Node是DOM当中最基本的节点，比Element还要更底层一些

# nodeType类型

节点类型 描述 子节点
1 Element 代表元素 Element, Text, Comment, ProcessingInstruction, CDATASection, EntityReference
2 Attr 代表属性 Text, EntityReference
3 Text 代表元素或属性中的文本内容。 None
4 CDATASection 代表文档中的 CDATA 部分（不会由解析器解析的文本）。 None
5 EntityReference 代表实体引用。 Element, ProcessingInstruction, Comment, Text, CDATASection, EntityReference
6 Entity 代表实体。 Element, ProcessingInstruction, Comment, Text, CDATASection, EntityReference
7 ProcessingInstruction 代表处理指令。 None
8 Comment 代表注释。 None
9 Document 代表整个文档（DOM 树的根节点）。 Element, ProcessingInstruction, Comment, DocumentType
10 DocumentType 向为文档定义的实体提供接口 None
11 DocumentFragment 代表轻量级的 Document 对象，能够容纳文档的某个部分 Element, ProcessingInstruction, Comment, Text, CDATASection, EntityReference
12 Notation 代表 DTD 中声明的符号。 None

重点：1，2，3，8，9

# Node常用属性

属性名/方法名 描述
nodeName 返回当前节点作为字符串的名称(大写),只读属性。
nodeValue 该属性返回或设置当前节点的值 , 对于文档本身,nodeValue返回null.对于text,comment节点,则返回节点的内容，对于属性节点返回属性的值。
childNodes 只读属性。返回NodeList的孩子nodes给定元素
ownerDocument 只读属性。返回当前节点的顶层的document对象，即整个文档的文档节点。 如果在 文档节点自身上使用此属性，则返回null
cloneNode() 方法返回调用该方法的节点一个副本 参数 :可选。表示是否采用深度克隆，如果true，则采用深度克隆(该节点的后代节点也都会被克隆);如果false,则只是克隆该节点本身，（注意这里不是“浅拷贝和深拷贝的意思”） 返回值：克隆生成的副本节点
